## 一、项目概述

### 1.1 项目背景

在快节奏的现代生活中，个人成长与自我管理变得尤为重要。本项目旨在打造一款**全方位的个人每日记录与目标管理系统**，帮助用户高效地进行日常规划、财务记账、知识摘录、目标追踪和数据分析，从而实现更有序、更高效的个人成长。

### 1.2 项目定位

- **核心理念**：以"日"为基本单位，向上聚合至周、月、年、五年维度，实现从微观到宏观的全链路个人管理
- **目标用户**：追求个人成长、注重自我管理的职场人士及学生群体
- **产品形态**：Web端（管理后台 + 用户前台）+ 微信小程序

### 1.3 技术架构总览

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 后端服务 | Java 17 + Spring Boot 3.2 | RESTful API 服务 |
| 数据库 | MySQL 8.0 | 主数据存储 |
| 缓存 | Redis 7 | 会话管理 & 热点数据缓存 |
| 前端Web | Vue 3 + TypeScript + Vite 5 | SPA 单页应用 |
| 小程序 | uni-app (Vue 3) | 微信小程序 |
| 对象存储 | MinIO / 阿里云 OSS | 图片与文件存储 |
| ORM框架 | MyBatis-Plus 3.5 | 数据库访问层 |
| 接口文档 | Knife4j (Swagger增强) | API文档自动生成 |

### 2.2 数据看板模块

#### 2.2.1 功能说明

数据看板是系统的**首页核心模块**，提供多维度的数据聚合与可视化展示。

#### 2.2.2 今日概览

| 数据卡片 | 展示内容 | 数据来源 |
|---------|---------|---------|
| 计划完成率 | 今日已完成/总计划数，进度环 | daily_plan |
| 记账汇总 | 今日收入/支出/结余 | daily_accounting |
| 摘录数量 | 今日新增摘录条数 | daily_excerpt |
| 总结状态 | 今日是否已完成总结 | daily_summary |
| 目标进度 | 本周/本月目标完成进度 | goal_plan |

#### 2.2.3 统计维度

- **本周统计**：计划完成趋势折线图、收支柱状图、摘录分类饼图
- **本月统计**：日历热力图、收支趋势分析、目标达成率雷达图
- **年度统计**：月度对比分析、年度收支报表、成长曲线图

#### 2.2.4 接口设计

```
GET /api/v1/dashboard/today          — 今日概览数据
GET /api/v1/dashboard/week           — 本周统计数据
GET /api/v1/dashboard/month          — 本月统计数据
GET /api/v1/dashboard/year           — 年度统计数据
GET /api/v1/dashboard/trend          — 趋势分析（支持自定义时间范围）
```

### 2.3 目标管理模块

#### 2.3.1 目标层级体系

```
五年计划（战略方向 & 人生目标）
  └── 年度计划（年度关键成果）
       └── 月度计划（月度里程碑）
            └── 周计划（周度执行目标）
                 └── 每日计划（日常任务分解）
```

#### 2.3.2 目标数据模型

每个目标包含以下核心属性：

| 字段 | 说明 | 示例 |
|------|------|------|
| 目标标题 | 简明描述 | "掌握系统设计" |
| 目标描述 | 详细说明 | 具体描述内容 |
| 目标类型 | 五年/年度/月度/周 | YEARLY |
| 目标分类 | 职业/学习/健康/财务/生活 | CAREER |
| 开始日期 | 计划开始时间 | 2026-01-01 |
| 截止日期 | 计划完成时间 | 2026-12-31 |
| 进度（%） | 0~100 | 45 |
| 状态 | 未开始/进行中/已完成/已放弃 | IN_PROGRESS |
| 父目标ID | 上级目标关联 | 可选 |
| 关键结果 | OKR中的KR列表 | JSON数组 |

#### 2.3.3 接口设计

```
POST   /api/v1/goals                   — 创建目标
GET    /api/v1/goals                   — 目标列表（支持类型/分类/状态筛选）
GET    /api/v1/goals/{id}              — 目标详情
PUT    /api/v1/goals/{id}              — 更新目标
DELETE /api/v1/goals/{id}              — 删除目标
PUT    /api/v1/goals/{id}/progress     — 更新目标进度
GET    /api/v1/goals/tree              — 目标树形结构（含子目标）
GET    /api/v1/goals/statistics        — 目标统计分析
```

### 2.4 每日计划模块

#### 2.4.1 功能说明

每日计划是系统的**高频使用模块**，支持任务的创建、分类、优先级管理和时间追踪。

#### 2.4.2 任务属性

| 字段 | 说明 | 规则 |
|------|------|------|
| 任务标题 | 任务简述 | 必填，最大100字符 |
| 任务描述 | 详细内容 | 可选，支持Markdown |
| 所属日期 | 任务归属日期 | 默认当天 |
| 优先级 | 紧急程度 | P0(最高) ~ P3(最低) |
| 分类标签 | 任务分类 | 工作/学习/生活/健康 |
| 预计时间 | 预估耗时（分钟） | 可选 |
| 实际时间 | 实际耗时（分钟） | 完成后填写 |
| 状态 | 执行状态 | 待办/进行中/已完成/已取消 |
| 关联目标 | 关联的周/月目标ID | 可选 |
| 排序权重 | 展示顺序 | 数字越小越靠前 |

#### 2.4.3 特色功能

- **任务模板**：支持保存常用任务为模板，快速创建
- **拖拽排序**：前端支持拖拽调整任务顺序
- **番茄钟集成**：支持按番茄钟模式记录任务时间
- **任务延期**：未完成任务可一键顺延至下一天
- **每日回顾**：自动统计当日任务完成率和时间分配

#### 2.4.4 接口设计

```
POST   /api/v1/daily-plans                      — 创建任务
GET    /api/v1/daily-plans                      — 任务列表（按日期查询）
GET    /api/v1/daily-plans/{id}                 — 任务详情
PUT    /api/v1/daily-plans/{id}                 — 更新任务
DELETE /api/v1/daily-plans/{id}                 — 删除任务
PUT    /api/v1/daily-plans/{id}/status          — 更新任务状态
PUT    /api/v1/daily-plans/batch-sort           — 批量更新排序
POST   /api/v1/daily-plans/{id}/postpone        — 任务延期
GET    /api/v1/daily-plans/statistics           — 任务统计
POST   /api/v1/daily-plans/templates            — 保存为模板
GET    /api/v1/daily-plans/templates            — 获取模板列表
```

### 2.5 每日摘录模块

#### 2.5.1 功能说明

记录每日阅读、学习过程中的**灵感、金句、知识点**，构建个人知识库。

#### 2.5.2 摘录属性

| 字段 | 说明 |
|------|------|
| 摘录内容 | 主体内容，支持富文本 |
| 来源 | 书籍/文章/视频/播客/其他 |
| 来源标题 | 出处名称 |
| 来源链接 | URL（可选） |
| 分类标签 | 多标签，如"技术"、"哲学" |
| 个人感悟 | 对摘录的个人思考 |
| 收藏标记 | 是否收藏 |
| 图片附件 | 支持上传图片 |

#### 2.5.3 特色功能

- **标签管理**：支持自定义标签体系，方便分类检索
- **全文搜索**：支持对摘录内容的全文模糊搜索
- **随机回顾**：随机展示历史摘录，温故知新
- **导出功能**：支持导出为Markdown/PDF格式

#### 2.5.4 接口设计

```
POST   /api/v1/excerpts                 — 创建摘录
GET    /api/v1/excerpts                 — 摘录列表（分页 + 筛选）
GET    /api/v1/excerpts/{id}            — 摘录详情
PUT    /api/v1/excerpts/{id}            — 更新摘录
DELETE /api/v1/excerpts/{id}            — 删除摘录
PUT    /api/v1/excerpts/{id}/favorite   — 切换收藏状态
GET    /api/v1/excerpts/random          — 随机获取摘录
GET    /api/v1/excerpts/tags            — 获取所有标签
GET    /api/v1/excerpts/search          — 全文搜索
```

### 2.6 每日记账模块

#### 2.6.1 功能说明

提供简洁高效的个人收支管理，支持**分类统计、预算管控、趋势分析**。

#### 2.6.2 账目属性

| 字段 | 说明 |
|------|------|
| 类型 | 收入 / 支出 |
| 金额 | 精确到分（BigDecimal） |
| 一级分类 | 餐饮/交通/购物/工资/理财等 |
| 二级分类 | 早餐/午餐/晚餐/零食等 |
| 备注 | 补充说明 |
| 记账日期 | 发生日期 |
| 记账时间 | 发生时间 |
| 账户 | 现金/微信/支付宝/银行卡 |
| 图片凭证 | 发票/截图等 |

#### 2.6.3 分类体系

```
支出分类：
├── 餐饮美食（早餐/午餐/晚餐/零食/饮品）
├── 交通出行（公交地铁/打车/加油/停车）
├── 购物消费（日用品/服饰/数码/家居）
├── 居住物业（房租/水电/物业/维修）
├── 娱乐休闲（电影/游戏/旅游/运动）
├── 医疗健康（门诊/药品/保健/体检）
├── 学习教育（书籍/课程/考试/培训）
├── 人情社交（礼金/聚餐/红包）
└── 其他支出

收入分类：
├── 工资薪酬（月薪/奖金/补贴）
├── 兼职收入
├── 理财收益（利息/股票/基金）
├── 转账收入
└── 其他收入
```

#### 2.6.4 预算管理

- 支持设置**月度总预算**和**分类预算**

GET    /api/v1/accounting/statistics/daily     — 日统计
GET    /api/v1/accounting/statistics/monthly   — 月统计
GET    /api/v1/accounting/statistics/yearly    — 年统计

## 三、非功能性设计

### 3.1 安全设计

| 安全项 | 方案 |
|--------|------|
| 认证方式 | JWT（Access Token 30min + Refresh Token 7天） |
| 密码存储 | BCrypt 加密存储 |
| 接口安全 | Spring Security + 权限注解 |
| 数据校验 | JSR 380（@Valid 参数校验） |
| SQL注入防护 | MyBatis-Plus 参数化查询 |
| XSS防护 | 富文本内容过滤（jsoup） |
| 接口限流 | 基于Redis的滑动窗口限流 |
| CORS策略 | 白名单机制 |

### 3.2 性能设计

- **Redis缓存**：热点数据（统计数据、用户信息）缓存，TTL 5~30分钟
- **分页查询**：所有列表接口强制分页，默认每页20条
- **数据库索引**：按查询频率建立组合索引（user_id + date 等）
- **懒加载**：前端路由懒加载，减少首屏加载时间
- **接口合并**：看板页面使用聚合接口，减少请求次数

### 3.3 统一响应格式

```json
{
  "code": 200,
  "message": "操作成功",
  "data": { },
  "timestamp": 1714200000000
}
```

### 3.4 错误码规范

| 错误码 | 含义 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未认证/Token过期 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |
| 1001~1999 | 用户模块业务错误 |
| 2001~2999 | 计划模块业务错误 |
| 3001~3999 | 记账模块业务错误 |
| 4001~4999 | 摘录模块业务错误 |
| 5001~5999 | 目标模块业务错误 |
| 6001~6999 | 总结模块业务错误 |

### 3.5 分页响应格式

```json
{
  "code": 200,
  "message": "查询成功",
  "data": {
    "records": [],
    "total": 100,
    "pageNum": 1,
    "pageSize": 20,
    "pages": 5
  }
}
```

## 四、部署架构

### 4.1 环境说明

| 环境 | 用途 | 说明 |
|------|------|------|
| 开发环境 | 本地开发调试 | localhost |
| 测试环境 | 功能验证 | Docker本地部署 |
| 生产环境 | 正式上线 | 云服务器部署 |

### 4.2 生产环境架构

```
用户 → CDN(静态资源) → Nginx → Vue3前端(:80)
                       Nginx → Spring Boot(:8080) → MySQL(:3306)
                                                  → Redis(:6379)
                                                  → MinIO(:9000)
```

### 4.3 Docker Compose 部署

所有服务通过 Docker Compose 编排，一键部署：mysql、redis、minio、backend、frontend、nginx。
