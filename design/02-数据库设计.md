## 一、设计规范

### 1.1 命名规范

- 表名：`t_模块_实体`，全小写下划线分隔
- 字段名：全小写下划线分隔（snake_case）
- 索引名：`idx_表名_字段名`
- 唯一索引：`uk_表名_字段名`

### 1.2 公共字段

每张表统一包含以下公共字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键，自增 |
| user_id | BIGINT | 用户ID，数据隔离 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |
| is_deleted | TINYINT(1) | 逻辑删除标记（0=正常，1=删除） |

### 1.3 字符集

- 字符集：`utf8mb4`
- 排序规则：`utf8mb4_general_ci`

## 三、表结构设计

### 3.1 用户表（t_user）

```sql
CREATE TABLE t_user (
    id            BIGINT AUTO_INCREMENT PRIMARY KEY,
    username      VARCHAR(50)  NOT NULL COMMENT '用户名',
    password      VARCHAR(200) NOT NULL COMMENT '密码(BCrypt)',
    nickname      VARCHAR(50)  DEFAULT NULL COMMENT '昵称',
    avatar        VARCHAR(500) DEFAULT NULL COMMENT '头像URL',
    email         VARCHAR(100) DEFAULT NULL COMMENT '邮箱',
    phone         VARCHAR(20)  DEFAULT NULL COMMENT '手机号',
    wx_openid     VARCHAR(100) DEFAULT NULL COMMENT '微信OpenID',
    wx_unionid    VARCHAR(100) DEFAULT NULL COMMENT '微信UnionID',
    signature     VARCHAR(200) DEFAULT NULL COMMENT '个人签名',
    status        TINYINT      DEFAULT 1 COMMENT '状态(1=正常,0=禁用)',
    last_login_at DATETIME     DEFAULT NULL COMMENT '最后登录时间',
    created_at    DATETIME     DEFAULT CURRENT_TIMESTAMP,
    updated_at    DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    is_deleted    TINYINT(1)   DEFAULT 0,
    UNIQUE KEY uk_user_username (username),
    UNIQUE KEY uk_user_wx_openid (wx_openid)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

### 3.2 用户偏好设置表（t_user_settings）

CREATE TABLE t_user_settings (
    id              BIGINT AUTO_INCREMENT PRIMARY KEY,

### 3.3 目标计划表（t_goal_plan）

```sql
CREATE TABLE t_goal_plan (
    id            BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id       BIGINT       NOT NULL COMMENT '用户ID',
    parent_id     BIGINT       DEFAULT NULL COMMENT '父目标ID',
    title         VARCHAR(200) NOT NULL COMMENT '目标标题',

    goal_type     VARCHAR(20)  NOT NULL COMMENT '目标类型(FIVE_YEAR/YEARLY/MONTHLY/WEEKLY)',

    start_date    DATE         NOT NULL COMMENT '开始日期',
    end_date      DATE         NOT NULL COMMENT '截止日期',
    progress      INT          DEFAULT 0 COMMENT '进度(0~100)',
    status        VARCHAR(20)  DEFAULT 'NOT_STARTED' COMMENT '状态(NOT_STARTED/IN_PROGRESS/COMPLETED/ABANDONED)',
    priority      INT          DEFAULT 0 COMMENT '优先级',
    sort_order    INT          DEFAULT 0 COMMENT '排序',
    created_at    DATETIME     DEFAULT CURRENT_TIMESTAMP,
    updated_at    DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    is_deleted    TINYINT(1)   DEFAULT 0,
    INDEX idx_goal_user_type (user_id, goal_type),
    INDEX idx_goal_user_status (user_id, status),
    INDEX idx_goal_parent (parent_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='目标计划表';
```

## 四、索引设计说明

### 4.1 核心查询场景与索引覆盖

| 查询场景 | 使用索引 | 说明 |
|---------|---------|------|
| 按日期查看每日计划 | idx_plan_user_date | 最高频查询 |
| 按日期查看记账 | idx_acc_user_date | 高频查询 |
| 按日期查看摘录 | idx_excerpt_user_date | 高频查询 |
| 按日期查看总结 | uk_summary_user_date | 唯一索引即可 |
| 目标树查询 | idx_goal_parent | 树形结构查询 |
| 按类型筛选目标 | idx_goal_user_type | 分类筛选 |
| 摘录全文搜索 | ft_excerpt_content | MySQL全文索引 |
| 分类统计记账 | idx_acc_category | 按分类聚合 |

### 4.2 分区策略（可选优化）

当数据量达到百万级时，可按 `accounting_date` / `plan_date` 进行**按月分区**：

```sql
ALTER TABLE t_accounting PARTITION BY RANGE (YEAR(accounting_date) * 100 + MONTH(accounting_date)) (
    PARTITION p202601 VALUES LESS THAN (202602),
    PARTITION p202602 VALUES LESS THAN (202603),
    ...
);
```

## 五、初始化数据

### 5.1 预设记账分类

系统需预置标准的收支分类数据，在应用首次启动时通过 Flyway/Liquibase 数据迁移脚本初始化。

**支出分类示例**：餐饮美食、交通出行、购物消费、居住物业、娱乐休闲、医疗健康、学习教育、人情社交、其他支出。每个一级分类下包含对应的二级分类。

**收入分类示例**：工资薪酬、兼职收入、理财收益、转账收入、其他收入。

---

> **说明**：所有表均采用 InnoDB 引擎，支持事务处理。逻辑删除字段 `is_deleted` 配合 MyBatis-Plus 全局配置实现软删除。
